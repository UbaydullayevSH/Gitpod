HTML(HYPER TEXT MARKUP LANGUAGE)

Frequently Asked Questions

What is HTML?

HTML (HyperText Markup Language) is the standard language used to create and structure web pages. It defines the content of a webpage using elements like headings, paragraphs, links, images, and multimedia.

What is the difference between a block-level element and an inline element in HTML?

A block-level element takes up the full width available, starting on a new line and stretching as far as its container allows.

What are semantic elements in HTML?

Semantic elements in HTML are those that clearly describe their meaning to both the browser and the developer.

CSS (CASCADING STYLE SHEET)

Frequently Asked Questions

What is the difference between class and id in CSS?

class is used to select multiple elements on a page, and it can be applied to any number of elements. It is selected in CSS using a period (.) followed by the class name. Example: .myClass. id is used to select a single unique element on a page. An id should be used only once per page. It is selected in CSS using a hash (#) followed by the id name. Example: #myId.

What is the box model in CSS?

The CSS box model describes how the elements on a webpage are structured. Every HTML element is considered as a box with the following properties: Content: The actual content of the element (e.g., text or images). Padding: The space between the content and the border. It adds internal spacing inside the box. Border: The edge of the element, which surrounds the padding (if any). Margin: The space outside the border. It adds external spacing between the element and surrounding elements.

What is the purpose of flexbox in CSS?

Flexbox (Flexible Box Layout) is a layout model in CSS that makes it easier to design flexible and responsive layouts. It allows elements to be arranged in rows or columns, and it provides the ability to distribute space dynamically between items, even when their sizes are unknown or dynamic.

JavaScript (JS)

Frequently Asked Questions

What is the difference between var, let, and const in JavaScript?

var: Declares a variable that is function-scoped or globally scoped. Variables declared with var can be re-declared and updated within their scope. let: Declares a block-scoped variable that can be updated but not re-declared within the same scope. const: Declares a block-scoped variable whose value cannot be re-assigned after its initial assignment.

What is a callback function in JavaScript?

A callback function is a function that is passed as an argument to another function and is executed after the completion of that function's execution. Callback functions are often used in asynchronous operations, such as handling events, reading files, or making HTTP requests.

What are promises in JavaScript?

A promise in JavaScript is an object representing the eventual completion (or failure) of an asynchronous operation. It allows you to write cleaner, more readable code when dealing with asynchronous tasks such as API requests, file handling, etc. A promise can be in one of three states: Pending: The operation is still in progress. Fulfilled: The operation completed successfully. Rejected: The operation failed.

React (JS)

Frequently Asked Questions

What is the difference between state and props in React?

State is a built-in object in React components that allows components to track and manage their internal data or state. It is mutable, meaning you can update it using the setState method (class components) or the useState hook (functional components). The state directly affects the component's behavior and rendering. Props (short for "properties") are read-only objects passed from a parent component to a child component. They allow data to flow from the parent to the child and are immutable in the child component. Props are used to pass information like data, functions, or event handlers to child components.

What are functional components and class components in React?

Functional components are simpler components written as JavaScript functions. They accept props as arguments and return JSX, which is rendered by React. Functional components were stateless before React introduced hooks, but now they can also manage state and side effects.

What is the use of the useEffect hook in React?

The useEffect hook in React is used to perform side effects in functional components. Side effects are operations that interact with the outside world, such as fetching data, updating the DOM, or setting up subscriptions. It replaces lifecycle methods such as componentDidMount, componentDidUpdate, and componentWillUnmount in class components.
